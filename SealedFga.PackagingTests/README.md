# SealedFga.PackagingTests (Tier D)

Consumes SealedFga **as the packed NuGet package** — the runtime library from `lib/net10.0`
plus the generator loaded from `analyzers/dotnet/cs`, exactly as it ships. Its job is to prove
the package is complete and works as shipped from one `PackageReference` alone: the generator
(and its bundled `OpenFga.Language.dll`) loads and runs from the analyzer payload, the package's
build props kick in (auto `*.fga` pickup + `LangVersion`), and the runtime library's EF model
wiring works against the consumer's EF Core.

This project is **not** part of `SealedFga.slnx`: a `PackageReference` can't express "build the
library first", so it must be packed into the local feed before it can restore.

## Running it

```bash
# 1. Pack SealedFga into the local feed this project restores from.
dotnet pack SealedFga/SealedFga.csproj -c Debug -o SealedFga.PackagingTests/local-packages

# 2. The package version is fixed (SealedFgaVersion in settings.props), so NuGet caches it by
#    version — clear the cached copy so a freshly packed build is picked up.
dotnet nuget locals global-packages --clear   # or: rm -rf ~/.nuget/packages/sealedfga

# 3. Restore + test.
dotnet test SealedFga.PackagingTests/SealedFga.PackagingTests.csproj -c Debug
```

`local-packages/` is git-ignored and regenerated by step 1.
